How it works
1
TreeWalker text scan — walks every text node in the DOM looking for "Sales Engineer Notes". This finds elements that querySelectorAll misses due to SFDC's rendering.
2
Skips ghost elements — SFDC keeps hidden zero-height duplicates in the DOM. The bookmarklet checks offsetHeight and skips any match with no dimensions.
3
Walks up to section — from the visible field label, climbs the DOM to the parent div.test-id__section so the whole section scrolls into view.
4
Scrolls and highlights — smooth scrolls the section to center screen with a brief blue outline that fades after 3 seconds.
Why this approach? SFDC Lightning renders opportunity pages with native window scrolling and keeps invisible duplicate elements in the DOM. Standard selectors like querySelectorAll miss the visible instance, and scroll containers have overflow: visible so programmatic scroll detection fails. The TreeWalker + offsetHeight check bypasses both issues. At 621 characters, it's well within all browser bookmarklet limits.
